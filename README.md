# КПІ: Розробка мобільних застосувань під Android (Ковтун)

Цей репозиторій містить приклади лабораторних робіт та завершений проєкт, створені в процесі вивчення розробки Android-застосунків.

## Структура папок 1–6

- **`1/MyApplication`**: проста форма замовлення квітів з полем введення назви букета, вибором кольору та діапазону ціни (RadioGroup) і відображенням зведення замовлення.
- **`2/lab2`**: робота з фрагментами — головна активність підвантажує `InputFragment`, який передає введені дані в `ResultFragment` (демонстрація навігації/обміну даними між фрагментами).
- **`3/lab3`**: розширена версія роботи з фрагментами та збереженням введеної інформації (окремі `InputFragment` і `ResultFragment`, додатковий екран/активність для перегляду збережених даних).
- **`4/labmedia`**: медіаплеєр на базі `ExoPlayer`, який уміє:
  - відтворювати відео з URL;
  - вибирати та програвати локальні аудіо- й відеофайли через `ActivityResultContracts.OpenDocument`;
  - показувати назву поточного файла та керувати відтворенням.
- **`5/compassapp`**: цифровий компас на основі сенсора `TYPE_ROTATION_VECTOR`:
  - обчислює азимут, застосовує фільтрацію значення;
  - обертає зображення циферблату;
  - показує числовий кут і текстовий напрямок (N, NE, E, ...).
- **`6/HabitTracker_FullProject`**: завершений застосунок для трекінгу звичок із локальною базою (Room), нотифікаціями, щоденними нагадуваннями та екраном статистики.

## Як запускати проєкти 1–6

1. В Android Studio відкрий потрібну папку як проєкт (наприклад, `1/MyApplication`, `2/lab2`, ...).
2. Дочекайся синхронізації Gradle.
3. Натисни кнопку запуску (`Run`) та обери емулятор або фізичний пристрій.

- `1/MyApplication`: запускає просту форму введення/вибору та показує сформоване «замовлення».
- `2/lab2`: стартує робота з фрагментами (`InputFragment` -> `ResultFragment`): введи дані та підтвердь, щоб побачити результат.
- `3/lab3`: аналогічно працює з фрагментами, але додатково зберігає введені дані у внутрішній файл `flowers.txt`; для перегляду відкрий `StorageActivity`.
- `4/labmedia`: для відтворення за URL потрібен інтернет; для програвання файлів використовується системний вибір (`OpenDocument`) (аудіо/відео).
- `5/compassapp`: працює зі сенсором орієнтації (краще тестувати на реальному пристрої, на емуляторі покази можуть бути неточні або відсутні).
- `6/HabitTracker_FullProject`: при першому запуску на Android 13+ буде запит на дозвіл `POST_NOTIFICATIONS` для щоденних нагадувань.

## Основний проєкт: HabitTracker

`6/HabitTracker_FullProject` — застосунок для ведення звичок із щоденними нагадуваннями та статистикою виконання.

### Що вміє

- Додавання звички (назва, опис, час нагадування) у `AddHabitActivity` за допомогою вибору часу.
- Головний екран показує список звичок (`RecyclerView`) і дозволяє відмічати виконання.
- Підрахунок серій (streak): поточної та найкращої, а також кількості виконаних днів.
- Щоденні нагадування через `AlarmManager` + `BroadcastReceiver` (`ReminderReceiver`) та відображення нотифікацій у `NotificationHelper`.
- Видалення звички з автоматичною відміною її майбутніх нагадувань.
- Статистика у вигляді стовпчикової діаграми за допомогою `MPAndroidChart` у `StatisticsActivity`.

### Технології

- Kotlin, AndroidX (ViewModel/LiveData), Material Components
- Room (локальне збереження даних)
- RecyclerView, NotificationCompat
- AlarmManager для щоденних нагадувань
- MPAndroidChart для візуалізації статистики

## Як запускати HabitTracker (папка 6)

1. В Android Studio відкрий потрібний модуль/папку (наприклад, `6/HabitTracker_FullProject`).
2. Зібрати проєкт і запустити на емуляторі або пристрої.
3. На Android 13+ при першому запуску буде запит на дозвіл `POST_NOTIFICATIONS` для роботи нагадувань.
